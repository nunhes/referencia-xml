CSS en XML

Las follas de estilo CSS se desenvolveron para utilizarse junto con o linguaxe de marcas HTML, pero un documento XML tamén puede hacer referencia a unha folla de estilo CSS e visualizarse no navegador.

O W3C aprobó en xuño de 1999 a recomendación Associating Style Sheets with XML documents 1.0, editada por James Clark e en outubro de 2010 aprobó a recomendación Associating Style Sheets with XML documents 1.0 (2ª edición), editada por James Clark, que definen cómo vincular documentos XML con follas de estilo CSS.

Documentos XML sin follas de estilo

Cuando un documento XML no enlaza a unha folla de estilo, os navegadores muestran o contenido completo do documento, etiquetas incluidas.

Por ejemplo, el documento seguinte:

<?xml version="1.0" encoding="UTF-8"?>
<biblioteca>
  <libro>
    <titulo>La vida está en otra parte</titulo>
    <autor>Milan Kundera</autor>
    <fechaPublicacion ano="1973"/>
  </libro>
  <libro>
    <titulo>Pantaleón e as visitadoras</titulo>
    <autor fechaNacimiento="28/03/1936">Mario Vargas Llosa</autor>
    <fechaPublicacion ano="1973"/>
  </libro>
  <libro>
    <titulo>Conversación na catedral</titulo>
    <autor fechaNacimiento="28/03/1936">Mario Vargas Llosa</autor>
    <fechaPublicacion ano="1969"/>
  </libro>
</biblioteca>

Documentos XML con follas de estilo

De acuerdo con a recomendación Associating Style Sheets with XML documents 1.0 (2ª edición), un documento XML puede enlazar a unha folla de estilo mediante a instrucción de procesamento <?xml-stylesheet ?>, de forma similar a como se hace en unha páxina web XHTML con a etiqueta <link />. En ambos casos o atributo href incluye o camino absoluto o relativo a a folla de estilo CSS.

La diferencia es que a etiqueta <link /> forma parte do encabezamiento (etiqueta <head>), mientras que a instrucción de procesamento <?xml-stylesheet ... ?> va al principio do documento, despois da declaración XML, como muestra o seguinte ejemplo:

Ejemplo de enlace a CSS en HTML5
Ejemplo de enlace a CSS en XML
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="utf-8" />
  <title>HTML 5</title>
  <link rel="stylesheet" type="text/css" href="estilo.css" />
</head>
<body>
  <p>Esta páxina es HTML 5 válido.</p>
</body>
</html>

<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet href="ejemplo.css"?>
<libro>
  <titulo>La vida está en otra parte</titulo>
  <autor>Milan Kundera</autor>
  <fechaPublicacion ano="1973"/>
</libro>

Si un documento XML enlaza a unha folla de estilo, os navegadores xa no muestran as etiquetas e aplican a folla de estilo.

  • En o ejemplo seguinte, se puede comprobar que o navegador xa no muestra as etiquetas, sino únicamente o texto contenido en ellas. e como a folla de estilo no contiene ninguna regla, todo o texto se ve da misma manera.
    Pero tamén se observa que o navegador muestra o texto dos dos elementos <titulo> e <autor> uno detrás de otro. Eso se debe a que o navegador aplica únicamente a folla de estilo enlazada e no aplica a folla de estilo predeterminada do navegador. Más adelante en esta misma sección, o apartado la propiedad display comenta cómo definir a forma de presentación dos elementos (como bloques, listas o tablas, por ejemplo).

Ejemplos de follas de estilo para XML

La propiedad display

Al escribir unha folla de estilo para un documento XML hay que tener en cuenta que os navegadores no aplican generalmente a folla de estilo predeterminada (salvo que se utilicen etiquetas html asociándolas al espacio de nombres do xhtml como en os ejemplos anteriores), así que al escribir unha folla de estilo para un documento XML hay que definir propiedades que normalmente no se suelen definir.

La máis importante es a propiedad display, que establece o modo de visualización do elemento.

Los valores posibles da propiedad display son none, block, compact, inline, inline-block, inline-table, list-item, marker(-), run-in, table, table-caption, table-cell, table-column, table-column-group, table-footer-group, table-header-group, table-row e table-row-group.

Los modos de visualización máis utilizados son:

  • none (ninguno) permite ocultar elementos.
  • block (bloque) es o modo de visualización dos elementos de tipo bloque, como párrafos (<p>), encabezados (<h1>, <h2>, ...), etc. O elemento ocupa toda a ventana en horizontal e ocupa o espacio vertical necesario para alojar o contenido do elemento.
  • inline (elemento en línea) es o modo de visualización de etiquetas como <strong>, <em>, <span>, <a>, etc. O elemento sólo ocupa o espacio necesario para alojar o contenido do elemento. En HTML os elementos en-línea teñen que estar contenidos dentro de elementos de bloque.
  • list-item (lista) es o modo de visualización de elementos de listas como <li>, <dd>, <dt>.
  • table (tabla), table-row (fila) e table-cell (celda) se combinan para mostrar elementos en forma de tabla.

Mostrar elementos como listas ordenadas

Si queremos mostrar elementos en forma de lista ordenada, hay que tener en cuenta que Firefox no muestra correctamente os contadores de as listas ordenadas:

CSS
Resultado
Enlace
titulo, autor, fechaPublicacion  {
  display: list-item;
  list-style: upper-alpha inside;
  margin-left: 0;
}

Nota: Firefox no aumenta o valor do contador, como se comenta na sección de diferencias entre navegadores.

Además, hay que tener en cuenta que os valores do contador aumentan en Chrome e Edge si os elementos con contadores se encuentran dentro do mismo elemento. En o ejemplo seguinte, o contador sólo se encuentra en os elementos <titulo> e por tanto o contador no aumenta, xa que os elementos <titulo> pertenecen a diferentes elementos <libro>.

CSS
Resultado
Enlace
titulo {
  display: list-item;
  list-style: upper-alpha inside;
  margin-left: 20px;
}

autor, fechaPublicacion {
  display: block;
  margin-left: 35px;
}

Nota: Aunque os tres navegadores se comportan da misma manera, Google Chrome e Edge no aumentan o contador porque no se debe hacer en este caso, mientras que Firefox no aumenta o valor do contador porque no lo hace nunca.

Si o contador se asocia al elemento <libro>, Chrome e Edge sí que aumentan o valor do contador, puesto que todos os elementos <libro> pertenecen al mismo elemento <biblioteca>.

CSS
Resultado
Enlace
libro {
  display: list-item;
  list-style: upper-alpha inside;
  margin-left: 20px;
}

titulo {
  display: inline;
}

autor, fechaPublicacion {
  display: block;
  margin-left: 20px;
}

Para conseguir que se vean as listas ordenadas en todos os navegadores, hay que utilizar contadores CSS, que se explican en la sección sobre pseudo-clases e pseudo-elementos dos apuntes de HTML e CSS:

CSS
Resultado
Enlace
biblioteca {
  counter-reset: contador;
}

libro:before {
  content: counter(contador, upper-alpha) ". ";
  counter-increment: contador;
}

titulo {
  display: inline;
  margin-left: 5px;
}

autor, fechaPublicacion {
  display: block;
  margin-left: 25px;
}

Al utilizar contadores, xa no necesario que os elementos con contador se encuentren contenidos no mismo elemento:

CSS
Resultado
Enlace
biblioteca {
  counter-reset: contador;
}

titulo:before {
  content: counter(contador, upper-alpha) ". ";
  counter-increment: contador;
}

titulo, autor, fechaPublicacion {
  display: block;
  margin-left: 25px;
}

autor, fechaPublicacion {
  margin-left: 45px;
}

Los atributos class e id

Cuando un documento XML incluye atributos class e id e a folla de estilo hace referencia a ellos mediante os selectores almohadilla (#) e punto (.), Firefox e Microsoft Edge aplican ambas reglas, pero Google Chrome aplica únicamente o selector almohadilla, como muestra o ejemplo seguinte:

XML
CSS
Resultado
Enlace
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet href="ejemplo.css"?>
<libro>
  <titulo class="color">La vida está en otra parte</titulo>
  <autor id="grande">Milan Kundera</autor>
  <fechaPublicacion ano="1973"/>
</libro>

.color {
  color: red;
}

#grande {
  font-size: 200%;
}
