Exemplo de documento XML

O marcado dun documento XML pode perfectamente parecer o marcado dunha páxina web, salvo que os nomes das etiquetas e atributos non son os da HTML.

<?xml version="1.0" encoding="UTF-8"?>
<paises>
  <pais>
    <nombre>España</nombre>
    <capital>Madrid</capital>
  </pais>
  <pais>
    <nombre>Francia</nombre>
    <capital>París</capital>
  </pais>
</paises>

Por suposto, non estamos suxeitos a unha estrutura concreta e a mesma información pódese gardar de diferentes modos:

<?xml version="1.0" encoding="UTF-8"?>
<paises>
  <pais nombre="Italia" capital="Roma" />
  <pais nombre="Francia" capital="París" />
</paises>

Conceptos e vocabulario

Documento XML
Un documento XML é un documento de texto plano (sen formato).
Procesador XML (XML processor) e aplicación (application)
Cando unha aplicación necesita ler un documento XML, a aplicación recurre a un procesador XML. O procesador XML (o analizador XML, en inglés XML parser) é o que lee o documento, analiza o contido e pasalle a información nun formato estruturado á aplicación. A recomendación XML especifica o que debe facer o procesador, pero non entra no que fai despois a aplicación con esa información.
Caracteres (characters)
Os documentos XML poden estar codificados en distintos xogos de caracteres (UTF-8, ISO-8859-1, etc).
Marcas (mark-up) e contido (content)
O texto que conten un documento XML se divide en marcas e contido. As marcas poden ser de dous tipos: etiquetas o referencias a entidades. Todo o que non son marcas é contido.
Elementos (elements)
Un elemento é unha compñente lóxica dun documento que ou ben comeza por unha etiqueta de apertura e termina por a etiqueta de peche correspondente o que consiste nunha única etiqueta baleira. O contido dun elemento é todo o que se atopa entre as etiquetas de apertura e peche, incluso se estes son tamén elementos en dito caso se deniminan elementos fillos.
Etiquetas (tags)
Unha etiqueta é un identificador que empieza polo carácter < e termina por >. Existen tres tipos de etiquetas:
  • as etiquetas de apertura (start-tag), que empezan polo carácter < e terminan por >. Por exemplo: 
    <apartado>
  • as etiquetas de peche (end-tag), que empezan polos caracteres </ e terminan por >. Por exemplo: 
    </apartado>
  • as etiquetas baleiras (empty tag), que empezan co carácter < e terminan por />. Por exemplo: 
    <linea />
Atributos (attributes)
Un atributo é un componente de as etiquetas que consiste en unha pareja nombre (name) / valor (value). Se puede encontrar nas etiquetas de apertura o nas etiquetas vacías, pero no nas de cierre. En unha etiqueta no puede haber dos atributos con o mismo nombre. La sintaxis é siempre nombreAtributo="valorAtributo". Por exemplo:
<profesor nombre="Bartolomé" apellidos="Sintes Marco" />
Comentarios (comments)
Un comentario é unha etiqueta que comienza por <!-- e acaba por -->. Los comentarios no pueden estar dentro de outras marcas e no pueden contener os caracteres "--". Dentro dun comentario as entidades de carácter no se reconocen, é decir, sólo se pueden utilizar os caracteres do xogo de caracteres do documento. Por exemplo:
<!-- Esto é un comentario -->
Declaración XML (XML declaration)
La declaración XML é unha etiqueta que comienza por <?xml e termina por ?> e que proporciona información sobre o propio documento XML. Aunque no é obligatoria é conveniente que aparezca, e debe aparecer siempre al principio do documento. No é unha instrucción de procesamento, pero tiene a misma sintaxis (empieza por <? e acaba por ?>).

La declaración xml indica o xogo de caracteres do documento. O xogo de caracteres que se utiliza en este curso é UTF-8:

<?xml version="1.0" encoding="UTF-8"?>

En XML, o nombre do xogo de caracteres se debe escribir en mayúsculas (UTF-8 en vez de utf-8).

La declaración xml tamén indica a versión XML utilizada. Aunque existe a versión XML 1.1, a versión máis común sigue siendo a versión XML 1.0.

Se pueden utilizar outros xogos de caracteres, como ISO-8859-1 (Europeo occidental):

<?xml version="1.0" encoding="ISO-8859-1"?>

Es importante que o xogo de caracteres que aparece na declaración sea o xogo de caracteres en que realmente está guardado o documento, porque si no o procesador XML puede tener problemas leyendo o documento.

<?xml version="1.0" encoding="UTF-8"?>
<!-- Este texto está guardado como UTF-8 e declara que é UTF-8 -->
<proba>
  <texto>á é í ó ú</texto>
</proba>

<?xml version="1.0" encoding="ISO-8859-1"?>
<!-- Este texto está guardado como UTF-8 sin BOM e declara que é ISO-8859-1 -->
<proba>
  <texto>á é í ó ú</texto>
</proba>

<?xml version="1.0" encoding="ISO-8859-1"?>
<!-- Este texto está guardado como ISO-8859-1 e declara que é ISO-8859-1 -->
<proba>
  <texto>á é í ó ú</texto>
</proba>

<?xml version="1.0" encoding="UTF-8"?>
<!-- Este texto está guardado como ISO-8859-1 e declara que é UTF-8 -->
<proba>
  <texto>á é í ó ú</texto>
</proba>

Nota:

En o caso do documento guardado como UTF-8 pero que declara ser ISO-8859-1, tamén le afecta que haya o no marca de orden de bytes (BOM).

La marca de orden de bytes son os primeros caracteres dun fichero UTF. Estos caracteres indican:

  • el orden dos bytes en os formatos de 16 o 32 bytes. Cuando a codificación dun carácter ocupa máis dun byte, os bytes que lo forman pueden estar no orden natural de lectura (big endian) o no contrario (little endian). Por exemplo o carácter A, cuyo código é 0041, se puede guardar en UTF-16 como 00 41 (big endian) o como 41 00 (little endian).
  • el formato do archivo en general. Para saber si un archivo está en formato UTF-8, 16 o 32 (es decir, si cada byte do archivo é un carácter, o os bytes se teñen que tomar de dos en dos, o de cuatro en cuatro), os programas sólo teñen que mirar os primeros bytes do archivo e reconocer a marca correspondiente.

En o caso dos archivos UTF-8, a marca de orden de bytes son os tres caracteres EF BB BF (que corresponden a os caracteres ï»¿). Pero como nun archivo UTF-8 no se necesita indicar o orden dos bytes, puesto que cada carácter ocupa solamente un byte, a marca é opcional, é decir, que hay archivos UTF-8 con BOM e archivos UTF-8 sin BOM. Los editores (como Eclipse PDT) no muestran estos caracteres y, suelen respetarlos al guardar de nuevo al archivo, pero outros (como o Bloc de notas de Windows) guardan os archivos UTF-8 siempre con BOM. Para ver esos caracteres, se puede utilizar un editor hexadecimal, que muestra todos os caracteres sin excepción.

O exemplo seguinte muestra como os navegadores son capaces de mostrar correctamente o contido si o archivo está guardado como UTF-8 con BOM. O motivo é que aunque o archivo declara que está guardado como ISO-8859-1, o navegador detecta o BOM e deduce que se trata dun archivo UTF-8:

<?xml version="1.0" encoding="ISO-8859-1"?>
<!-- Este texto está gardado como UTF-8 con BOM e declara que é ISO-8859-1 -->
<proba>
  <texto>á é í ó ú</texto>
</proba>

O exemplo seguinte mostra como os navegadores non son capaces de mostrar correctamente o contido si o archivo está guardado como UTF-8 sin BOM. O archivo se muestra como si fuera ISO-8859-1 xa que o archivo declara que está guardado como ISO-8859-1 e o navegador no detecta un BOM que le indique que realmente está guardado como UTF-8:

<?xml version="1.0" encoding="ISO-8859-1"?>
<!-- Este texto está guardado como UTF-8 sin BOM e declara que é ISO-8859-1 -->
<proba>
  <texto>á é í ó ú</texto>
</proba>
Definición de tipo de documento (DTD, Document Type Definition)
Una DTD é un documento que define a estrutura dun documento XML: os elementos, atributos, entidades, notacións, etc, que pueden aparecer, o orden e o número de veces que pueden aparecer, cuáles pueden ser hijos de cuáles, etc. O procesador XML a utiliza para verificar si un documento é válido, é decir, si o documento cumple as regras do DTD.
Declaración de tipo de documento (DOCTYPE, Document type declaration)
Una declaración de tipo de documento é unha etiqueta que comienza por <!DOCTYPE e acaba por > e que indica la(s) DTD(s) que debe utilizar o procesador XML para validar o documento. La DTD puede estar incluida no propio documento o ser un documento externo. Por exemplo, o seguinte exemplo muestra a declaración de tipo de documento que se debe incluir en os documentos XHTML 1.0 de tipo strict (en este caso, a DTD é un documento externo):
 <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
Instruccións de procesamento (PI, processing instruction)
Una instrucción de procesamento en unha etiqueta que empieza por "<?" e acaba por "?>" e que contiene instruccións dirigidas a as aplicacións que leen o documento. Pueden aparecer en calquera lugar do documento. Por exemplo:
<?xml-stylesheet type="text/xsl" href="estilo.xsl" ?>
Entidades de carácter
En XML se utilizan varias entidades de carácter de HTML, para poder escribir en cadenas de texto os caracteres que delimitan as marcas o as cadenas de texto :
Referencia a entidad Carácter
&lt; <
&gt; >
&amp; &
&apos; '
&quot; "
Seccións CDATA (CDATA section)
Una sección CDATA é unha etiqueta que comienza por <![CDATA[ e termina por ]]> e cuyo contido o procesador XML no interpreta como marcas sino como texto. é decir, que si aparecen os caracteres especiales (< & " ') en unha sección CDATA, o procesador XML no interpreta que empieza unha marca, sino que lo considera un carácter más. Se suele utilizar en documentos en os que aparecen muchas veces esos caracteres especiales para no tener que estar utilizando as referencias a entidades (&lt; e &amp;) que dificultan a lectura do documento. 
<?xml version="1.0" encoding="UTF-8"?>
<proba>
  <texto>Os caracteres < e & non poden escribirse
  si non é como comenzo de marcas</texto>
</proba>

<?xml version="1.0" encoding="UTF-8"?>
<proba>
  <texto>Os caracteres &lt; e &amp; non poden escribirse
  si non é como comenzo de marcas</texto>
</proba>

<?xml version="1.0" encoding="UTF-8"?>
<proba>
  <texto><![CDATA["Los caracteres < e & no pueden escribirse
  si no é como comienzo de marcas"]]></texto>
</proba>
Referencias a entidades
En XML se pueden definir entidades e utilizarlas como se utilizan as entidades de carácter en a HTML. unha entidad se define mediante unha etiqueta que comienza por <!ENTITY e termina por > e contiene o nombre e o valor da entidad. Por exemplo:

Como as entidades de carácter da HTML, para hacer referencia a unha entidad se escribe sin espacios intermedios o carácter "&", o nombre da entidad e o carácter ";". Al abrir o documento XML o procesador sustituye a referencia a a entidad por su valor. Por exemplo:

<?xml version="1.0" encoding="ISO-8859-1"?>
<!DOCTYPE autor [
  <!ENTITY yo "Bartolomé Sintes Marco">
  <!ELEMENT autor (#PCDATA)>
]>
<autor>&yo;</autor>

Documentos ben formados

Un documento XML debe estar ben formado, é decir debe cumprir as regras de sintaxis da recomendación XML. Para que un documento esté ben formado, al menos debe cumprir os seguintes puntos:

  • O documento contiene únicamente caracteres Unicode válidos.
  • Hay un elemento raíz que contiene al resto de elementos.
  • Los nombres dos elementos e de sus atributos no conteñen espacios.
  • O primer carácter dun nombre de elemento o de atributo puede ser unha letra, dos puntos (:) o subrayado (_).
  • O resto de caracteres pueden ser tamén números, guións (-) o puntos (.).
  • Los caracteres "<" e "&" sólo se utilizan como comienzo de marcas.
  • Las etiquetas de apertura, de cierre e vacías están correctamente anidadas (no se solapan) e no falta ni sobra ninguna etiqueta de apertura o cierre.
  • Las etiquetas de cierre coinciden con as de apertura (incluso no uso de mayúsculas e minúsculas).
  • Las etiquetas de cierre no conteñen atributos.
  • Ninguna etiqueta tiene dos atributos con o mismo nombre.
  • Todos os atributos teñen algún valor.
  • Los valores dos atributos están entre comillas (simples o dobles).
  • No existen referencias en os valores dos atributos.

Si un documento XML no está ben formado, no é un documento XML. Los procesadores XML deben rechazar calquera documento que contenga errores.

Documentos válidos

Un documento XML ben formado puede ser válido. Para ser válido, un documento XML debe:

  • incluir unha referencia a unha gramática,
  • incluir únicamente elementos e atributos definidos na gramática,
  • cumprir as regras gramaticais definidas na gramática.

Existen varias formas de definir unha gramática para documentos XML, as máis empregadas son :

  • DTD (Document Type Definition = Definición de Tipo de Documento). é o modelo máis antiguo, heredado do SGML.
  • XML Schema. É un modelo creado polo W3C como sucesor das DTDs.
  • Relax NG. É un modelo creado por OASIS, máis sinxelo que XML Schema.
