Breve historia da XML

As primeiras linguaxes de marcas se idearon a finais dos anos 60 do século XX, co cometido de estandarizar a estrutura de determinados documentos de texto (William W. Tunnicliffe, 1967). Nos anos 70 creáronse varias normas para linguaxes de marcas (GenCode, GML) para o seu uso na industria editorial. Xa en 1986, baixo a dirección do daquela investigador de IBM, Charles Goldfarb, creóuse SGML (Standard Generalized Markup Language), unha linguaxe estandarizada de marcado ben complexa que foi publicada como norma ISO 8879 ese mesmo ano.

En 1989, Tim Berners-Lee baseóuse na sintaxis da SGML para crear a HTML, a linguaxe de marcado da web. Inicialmente pensada para o seu uso no CERN, a web extendéuse por todo o planeta dunha forma fulminante. Para tutelar o crecemento e organización da web, en 1994 creóuse o W3C, un organismo internacional formado por empresas, institucións e investigadores.

O primeiro traballo do W3C foi normalizar a HTML, intentando incorporar as novidades que aparecían continuamente nos navegadores. A pesares diso, o crecemento da web seguía aumentado a urxencia de ampliar a HTML para todo tipo de usos. O W3C decidiu que a solución non podía ser seguir ampliando a HTML, senon crear unhas regras xerais para que calquera poidera crear linguaxes de marcas axeitados as súas necesidades, pero mantendo unhas estruturas e sintaxes comúns que permitiran compatibilizalos e tratalos coas mesmas ferramentas. Ese conxunto de regras é XML (eXtensible Markup Language = Linguaxe de Marcas Extensible), da que se publicou a primeira versión en 1998. Así, a XML non é unha linguaxe de marcas, pero si a base común elexida polo W3C para os seus desenvolvementos posteriores.

Nos primeiros anos do século XXI o W3C planteou unha triple líña de traballo:

  • desenvolver a XML, creando todo tipo de tecnoloxías para procesar documentos XML que poideran aplicarse a calquera linguaxe de marcas baseado en XML,
  • como a HTML dos anos 90 no cumplía as normas do XML xa que a HTML era anterior al XML, o W3C aprobou no ano 2000 a XHTML, unha versión da HTML que sí que cumplía as regras do XML, poñendo fin ao desenvolvemento da HTML.
  • poñer en marcha a iniciativa da Web semántica para dotar de "intelixencia" a a web, baseándose en XML.

Vinte anos despois, este ambicioso plan fracasou. Aínda que o W3C cumpriu a súa parte elaborando e aprobando decenas de recomendacións nas tres liñas de traballo, o mercado non as adoptou, especialmente na web.

Para empezar, os fabricantes de navegadores non aceptaron a XHTML e crearon en 2004 unha nueva organización, o WHATWG, para continuar o desenvolvementoda HTML que o W3C había abandonado. En 2007 o W3C reconoció su fracaso e retomó o desenvolvementoda HTML (incluyendo en él unha versión XHTML, basada en XML). En outubro de 2014 o W3C publicou a recomendación HTML 5, que incluye as dos variantes: HTML 5, no basada en XML, e XHTML 5, basada en XML (una tercera variante, llamada HTML políglota ha sido abandonada). Desde entonces se han ido publicando versións actualizadas da HTML 5, pero realmente o desenvolvementoda HTML xa no se hace no seno do W3C, sino no WHATWG.

Por otro lado, aunque por unos anos o XML se convirtió na ferramenta básica para o intercambio e almacenamento de información en moitos sectores da informática, pouco a pouco o mercado foi evolucionando noutras direccións, empregando tecnoloxías máis sinxelas non baseadas en XML.

Para terminar, a iniciativa da Web semántica (aínda supoñendo que tivera proposto as tecnoloxías axeitadas para permitir o procesamento intelixente da información contida nas páxinas web) non tivo a menor oportunidade de éxito, xa que a web non adoptou ningún dos compoñentes da web semántica.

Recoñecendo o fracaso, en 2013 o W3C xa cerrara prácticamente todos os grupos de traballo dedicados á web semántica e en 2018 o W3C tamén cerrou prácticamente todos os grupos de traballo dedicados á XML. Probablemente o principal motivo do fracaso sexa que XML, coa súa exixencia de ausencia de erros nos documentos, resulta demasiado estricto para unha web que en gran parte aínda se elabora manualmente.

Pero aínda que a web abandonara a XML, XML se segue utilizando en moitos ámbitos especializados para o almacenamento e transmisión da información, polo que costa creer que o seu desenvolvemento neses ámbitos non sexa retomado, dentro ou fora do W3C, nos próximos anos.

Tecnoloxías XML

O W3C e outras organizacións de normalización desenvolveron numerosas tecnoloxías XML. Nas seccións Normas e recomendacións XML e Web semántica se listan as recomendacións do W3C nas que se definen esas tecnoloxías.

O seguinte cadro mostra algunhas desas tecnoloxías agrupadas por temas:

As tecnoloxías XML básicas desenvoltas polo W3C son:

XML Namespaces (Espazos de nomes XML)
Define os mecanismos para permitir que nun mesmo documento se poidan utilizar elementos e atributos de diferentes linguaxes de marcas (resolvendo, por exemplo, os problemas de que algúns nomes dos elementos coincidan).
XML Infoset
Describe un modelo de datos abstracto para documentos XML a partir de elementos de información. Se utiliza nas especificacións de linguaxes XML, para describir restriccións na linguaxe.
XPath
Permite seleccionar os componentes dun documento XML e facilitar su acceso a os programas que procesan documentos XML.
XSLT
Linguaxe de transformación de documentos XML a outros formatos (XML o no XML)
XSL Formatting Objects (XSL-FO)
Linguaxe de marcas para formatear documentos XML que se usa, por exemplo, para xerar PDFs.
XQuery
Linguaxe de consulta orientado a XML, que permite acceder, manipular e devolver fragmentos de documentos XML.
XML Signature
Define a sintaxis e as regras de procesamento para crear firmas dixitais en documentos XML.
XML Encryption
Define a sintaxis e as regras de procesamento para encriptar documentos XML.

Outras tecnoloxías XML desenvoltas polo W3C non tiveron apenas implantación:

XInclude
Define o mecanismo para incluir un documento de texto (XML ou non XML) dentro dun documento XML.
XLink
Define métodos para incluir enlaces internos e externos en documentos XML e asociar metadatos aos enlaces.
XPointer
Define o mecanismo para identificar fragmentos dun documento.
