Qué es unha DTD

Una DTD es un documento que define a estrutura dun documento XML: os elementos, atributos, entidades, notacións, etc, que pueden aparecer, o orden e o número de veces que pueden aparecer, cuáles pueden ser hijos de cuáles, etc. O procesador XML utiliza a DTD para verificar si un documento es válido, es decir, si o documento cumple as regras do DTD.

Referencia a unha DTD nun documento XML

La DTD que debe utilizar o procesador XML para validar o documento XML se indica mediante a etiqueta DOCTYPE. La DTD puede estar incluida no propio documento, ser un documento externo o combinarse ambas.

  • La DTD puede incluirse no propio documento, con a seguinte sintaxis: 
    <!DOCTYPE nombre [
 ... declaracións ...
]>
  • La DTD puede estar nun documento externo y, si sólo va a ser utilizada por unha única aplicación, a sintaxis es a seguinte: 
    <!DOCTYPE nombre SYSTEM "uri">

    Se puede combinar unha DTD externa con unha DTD interna, con a seguinte sintaxis:

    <!DOCTYPE nombre SYSTEM "uri" [
 ... declaracións ...
]>
  • La DTD puede estar nun documento externo y, si va a ser utilizada por varias aplicacións, a sintaxis es a seguinte: 
    <!DOCTYPE nombre PUBLIC "fpi" "uri">

    Se puede combinar unha DTD externa con unha DTD interna, con a seguinte sintaxis:

    <!DOCTYPE nombre PUBLIC "fpi" "uri" [
 ... declaracións ...
]>

En todos estos casos:

  • "nombre" es o nombre do tipo de documento XML, que debe coincidir con o nombre do elemento raíz do documento XML.
  • "uri" es o camino (absoluto o relativo) hasta a DTD.
  • "fpi" es un identificador público formal (Formal Public Identifier).

Declaracións

Las DTDs describen a estrutura dos documentos XML mediante declaracións. Hay cuatro tipos de declaracións:

  • Declaracións de entidades
  • Declaracións de notacións
  • Declaracións de elementos, que indican os elementos permitidos nun documento e su contenido (que puede ser simplemente texto u outros elementos).
  • Declaracións de atributos, que indican os atributos permitidos en cada elemento e o tipo o valores permitidos de cada elemento.

Declaración de entidades

Una entidad consiste nun nombre e su valor (son similares a as constantes en os linguaxes de programación). Con algunhas excepciones, o procesador XML sustituye as referencias a entidades por sus valores antes de procesar o documento. unha vez definida a entidad, se puede utilizar no documento escribiendo unha referencia a a entidad, que empieza con o carácter "&", sigue con o nombre da entidad e termina con ";". (es decir, &nombreEntidad;)

Las entidades pueden ser internas o externas e tanto unhas como outras pueden ser xerais o paramétricas.

Las declaracións de entidades internas (xerais) siguen a seguinte sintaxis:

<!ENTITY nombreEntidad "valorEntidad">

En as declaracións de entidades externas (xerais) se distinguen dos casos:

  • La entidad hace referencia a un fichero de texto e en ese caso a entidad se sustituye por o contenido do archivo.

    La entidad puede ser unha entidad de sistema, con a seguinte sintaxis:

    <!ENTITY nombreEntidad SYSTEM "uri">

    o puede ser unha entidad pública, con a seguinte sintaxis:

    <!ENTITY nombreEntidad PUBLIC "fpi" "uri">
  • La entidad hace referencia a un fichero que no es de texto (por ejemplo, unha imagen) e en ese caso a entidad no se sustituye por o contenido do archivo.

    La entidad puede ser unha entidad de sistema, con a seguinte sintaxis:

    <!ENTITY nombreEntidad SYSTEM "uri" NDATA tipo>

    o puede ser unha entidad pública, con a seguinte sintaxis:

    <!ENTITY nombreEntidad PUBLIC "fpi" "uri" NDATA tipo>

En todos estos casos:

  • "nombreEntidad" es o nombre da entidad.
  • "valorEntidad" es o valor da entidad.
  • "uri" es o camino (absoluto o relativo) hasta un archivo.
  • "tipo" es o tipo de archivo (gif, jpg, etc).
  • "fpi" es un identificador público formal (Formal Public Identifier).

Las declaracións de entidades paramétricas siguen a mismas sintaxis que as xerais, pero llevan o carácter "%" antes do nombre da entidad. Por ejemplo:

<!ENTITY % nombreEntidad "valorEntidad">
<!ENTITY % nombreEntidad SYSTEM "uri">
<!ENTITY % nombreEntidad SYSTEM "uri" NDATA tipo>

La diferencia entre entidades xerais e paramétricas es que as entidades paramétricas se sustituyen por su valor en todo o documento (incluso na propia declaración de tipo de documento) mientras que as xerais no se sustituyen na declaración de tipo de documento.

Declaración de notacións

Las notacións se usan en XML para definir as entidades externas que no va a analizar o procesador XML (aunque sí lo hará a aplicación que trate un documento). Para hacer referencia estas entidades no se utiliza a notación habitual (&nombreEntidad;), sino que se utiliza o nombre da entidad directamente.

Declaración de elementos

Las declaracións dos elementos siguen a seguinte sintaxis:

<!ELEMENT nombreElemento (contenido)>

en a que "nombreElemento" es o nombre do elemento, e "(contenido)" unha expresión que describe o contenido do elemento.

Para definir o contenido do elemento se pueden utilizar os términos EMPTY, (#PCDATA) o ANY o escribir expresións máis complejas:

  • EMPTY: significa que o elemento es vacío, es decir, que no puede tener contenido. Los elementos vacíos pueden escribirse con etiquetas de apertura e cierre sin nada entre ellos, ni siquiera espacios, o con unha etiqueta vacía. EMPTY debe escribirse sin paréntesis. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo EMPTY>
]>

    <ejemplo></ejemplo>
    <ejemplo />
    <ejemplo>Esto es un ejemplo</ejemplo>                     <!-- ERROR: contiene texto -->

    <ejemplo><a></a></ejemplo>                                <!-- ERROR: contiene un elemento <a> -->
  • (#PCDATA): significa que o elemento puede contener texto. #PCDATA debe escribirse entre paréntesis. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo (#PCDATA)>
]>

    <ejemplo />
    <ejemplo>Esto es un ejemplo</ejemplo>
    <ejemplo><a></a></ejemplo>                                <!-- ERROR: contiene un elemento <a> -->
  • ANY: significa que o elemento puede contener calquera cosa (texto e outros elementos). ANY debe escribirse sin paréntesis. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo ANY>
  <!ELEMENT a ANY>
]>

    <ejemplo />
    <ejemplo>Esto es un ejemplo</ejemplo>
    <ejemplo>Esto es <a>un ejemplo</a></ejemplo>

Para indicar que un elemento puede o debe contener outros elementos se deben indicar os elementos, utilizando os conectores e modificadores seguintes:

  • , (coma): significa que o elemento contiene os elementos no orden indicado. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo (a, b)>
  <!ELEMENT a EMPTY>
  <!ELEMENT b EMPTY>
]>

    <ejemplo><a /><b /></ejemplo>
    <ejemplo><a /><b /><c /></ejemplo>                        <!-- ERROR: contiene un elemento <c /> -->
    <ejemplo><a /></ejemplo>                                  <!-- ERROR: falta o elemento <b /> -->
    <ejemplo><b /><a /></ejemplo>                             <!-- ERROR: o orden no es correcto -->
  • | (o lóxico): significa que o elemento contiene uno dos dos elementos. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo (a | b)>
  <!ELEMENT a EMPTY>
  <!ELEMENT b EMPTY>
]>

    <ejemplo><a /></ejemplo>
    <ejemplo><b /></ejemplo>
    <ejemplo><a /><b /></ejemplo>                              <!-- ERROR: están os dos elementos -->
    <ejemplo></ejemplo>                                        <!-- ERROR: no hay ningún elemento -->
  • ?: significa que o elemento puede aparecer o no, pero sólo unha vez. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo (a, b?)>
  <!ELEMENT a EMPTY>
  <!ELEMENT b EMPTY>
]>

    <ejemplo><a /></ejemplo>
    <ejemplo><a /><b /></ejemplo>
    <ejemplo><b /></ejemplo>                                  <!-- ERROR: falta o elemento <a /> -->
    <ejemplo><b /><b /></ejemplo>                             <!-- ERROR: o elemento <b /> aparece dos veces -->
  • *: significa que o elemento puede no aparecer o aparecer unha o máis veces. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo (a*, b)>
  <!ELEMENT a EMPTY>
  <!ELEMENT b EMPTY>
]>

    <ejemplo><b /></ejemplo>
    <ejemplo><a /><b /></ejemplo>
    <ejemplo><a /><a /><b /></ejemplo>
    <ejemplo><b /><a /></ejemplo>                             <!-- ERROR: o elemento <a /> aparece despois de <b /> -->
  • +: significa que o elemento tiene que aparecer unha o máis veces (no puede no aparecer). 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo (a+, b)>
  <!ELEMENT a EMPTY>
  <!ELEMENT b EMPTY>
]>

    <ejemplo><a /><b /></ejemplo>
    <ejemplo><a /><a /><b /></ejemplo>
    <ejemplo><b /></ejemplo>                                  <!-- ERROR: falta o elemento <a /> -->
  • (): permite agrupar expresiones. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo (a, (a|b))>
  <!ELEMENT a EMPTY>
  <!ELEMENT b EMPTY>
]>

    <ejemplo><a /><a /></ejemplo>
    <ejemplo><a /><b /></ejemplo>
    <ejemplo><a /></ejemplo>                                  <!-- ERROR: falta o elemento <a /> o <b /> -->
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo ((a, b)|(b, a))>
  <!ELEMENT a EMPTY>
  <!ELEMENT b EMPTY>
]>

    <ejemplo><a /><b /></ejemplo>
    <ejemplo><b /><a /></ejemplo>
    <ejemplo><a /><a /></ejemplo>                             <!-- ERROR: sólo admite <a /><b /> o <b /><a /> -->

Declaración de atributos

Una declaración de atributos sigue a seguinte sintaxis:

<!ATTLIST nombreElemento nombreAtributo tipoAtributo valorInicialAtributo >

en a que:

  • "nombreElemento" es o nombre do elemento para o que se define un atributo.
  • "nombreAtributo" es o nombre do atributo.
  • "tipoAtributo" es o tipo de datos.
  • "valorInicialAtributo" es o valor predeterminado do atributo (aunque tamén puede indicar outras cosas).

Para definir varios atributos dun mismo elemento, se puede utilizar unha o varias declaracións de atributos. Los seguintes ejemplos son equivalentes:

<!ATTLIST nombreElemento nombreAtributo1 tipoAtributo1 valorInicialAtributo1>
<!ATTLIST nombreElemento nombreAtributo2 tipoAtributo2 valorInicialAtributo2>

<!ATTLIST nombreElemento
  nombreAtributo1 tipoAtributo1 valorInicialAtributo1
  nombreAtributo2 tipoAtributo2 valorInicialAtributo2
 >

Los tipos de atributos son os seguintes:

  • CDATA: o atributo contiene caracteres (sin restricciones). 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo EMPTY>
  <!ATTLIST ejemplo color CDATA #REQUIRED>
]>

    <ejemplo color="" />
    <ejemplo color="amarillo" />
    <ejemplo color="azul marino #000080" />
    <ejemplo />                           <!-- ERROR: falta o atributo "color", obligatorio debido al #REQUIRED -->
    <ejemplo sabor="dulce" />             <!-- ERROR: o atributo "sabor" no está definido -->
  • NMTOKEN: o atributo sólo contiene letras, dígitos, e os caracteres punto ".", guion "-", subrayado "_" e dos puntos ":". 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo EMPTY>
  <!ATTLIST ejemplo color NMTOKEN #REQUIRED>
]>

    <ejemplo color="" />
    <ejemplo color="azul-marino" />
    <ejemplo color="1" />
    <ejemplo color="azul marino" />                   <!-- ERROR: hay un espacio en blanco -->
    <ejemplo color="#F0F0F0" />                       <!-- ERROR: contiene o carácter # -->
  • NMTOKENS: o atributo sólo contiene letras, dígitos, e os caracteres punto ".", guion "-", subrayado "_", dos puntos ":" (como o tipo NMTOKEN) e tamén espacios en blanco. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo EMPTY>
  <!ATTLIST ejemplo color NMTOKENS #REQUIRED>
]>

    <ejemplo color="" />
    <ejemplo color="1" />
    <ejemplo color="azul marino" />
    <ejemplo color="2*2" />                           <!-- ERROR: hay un asterisco -->
  • valores: o atributo sólo puede contener uno dos términos de unha lista. La lista se escribe entre paréntesis, con os términos separados por unha barra vertical "|". 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo EMPTY>
  <!ATTLIST ejemplo color (azul|blanco|rojo) #REQUIRED>
]>

    <ejemplo color="" />
    <ejemplo color="azul" />
    <ejemplo color="verde" />                         <!-- ERROR: "verde" no está na lista de valores -->
  • ID: o valor do atributo (no o nombre) debe ser único e no se puede repetir en outros elementos o atributos. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo (libro*)>
  <!ELEMENT libro (#PCDATA) >
  <!ATTLIST libro codigo ID #REQUIRED>
]>

    <ejemplo>
  <libro codigo="L1">Poema de Gilgamesh</libro>
  <libro codigo="L2">Los preceptos de Ptah-Hotep</libro>
</ejemplo>

    <ejemplo>
  <libro codigo="1">Poema de Gilgamesh</libro>            <!-- ERROR: o valor dun atributo de tipo ID no puede empezar con un número -->
  <libro codigo="L2">Los preceptos de Ptah-Hotep</libro>
</ejemplo>

    <ejemplo>
  <libro codigo="L1">Poema de Gilgamesh</libro>
  <libro codigo="L1">Los preceptos de Ptah-Hotep</libro>  <!-- ERROR: no se puede repetir un atributo de tipo ID -->
</ejemplo>
  • IDREF: o valor do atributo debe coincidir con o valor do atributo ID de otro elemento. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo ((libro|prestamo)*)>
  <!ELEMENT libro (#PCDATA) >
  <!ATTLIST libro codigo ID #REQUIRED>
  <!ELEMENT prestamo (#PCDATA) >
  <!ATTLIST prestamo libro IDREF #REQUIRED>
]>

    <ejemplo>
  <libro codigo="L1">Poema de Gilgamesh</libro>
  <prestamo libro="L1">Numa Nigerio</prestamo>
</ejemplo>

    <ejemplo>
  <libro codigo="L1">Poema de Gilgamesh</libro>
  <prestamo libro="L2">Numa Nigerio</prestamo>            <!-- ERROR: o valor "L2" no es ID de ningún elemento -->
</ejemplo>
  • IDREFS: o valor do atributo es unha serie de valores separados por espacios que coinciden con o valor do atributo ID de outros elementos. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo ((libro|prestamo)*)>
  <!ELEMENT libro (#PCDATA) >
  <!ATTLIST libro codigo ID #REQUIRED>
  <!ELEMENT prestamo (#PCDATA) >
  <!ATTLIST prestamo libro IDREFS #REQUIRED>
]>

    <ejemplo>
  <libro codigo="L1">Poema de Gilgamesh</libro>
  <libro codigo="L2">Los preceptos de Ptah-Hotep</libro>
  <prestamo libro="L1 L2">Numa Nigerio</prestamo>
</ejemplo>

    <ejemplo>
  <libro codigo="L1">Poema de Gilgamesh</libro>
  <libro codigo="L2">Los preceptos de Ptah-Hotep</libro>
  <prestamo libro="L1">Numa Nigerio</prestamo>
</ejemplo>

    <ejemplo>
  <libro codigo="L1">Poema de Gilgamesh</libro>
  <libro codigo="L2">Los preceptos de Ptah-Hotep</libro>
  <prestamo libro="L3">Numa Nigerio</prestamo>            <!-- ERROR: o valor "L3" no es ID de ningún elemento -->
</ejemplo>
  • ENTITY: o valor do atributo es algunha entidad definida na DTD.
  • ENTITIES: o valor do atributo es algunha de as entidades de unha lista de entidades definida na DTD.
  • NOTATION: o valor do atributo es algunha notación definida na DTD.

Los valores iniciales dos atributos son os seguintes:

  • #REQUIRED: o atributo es obligatorio, aunque no se especifica ningún valor predeterminado. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo EMPTY>
  <!ATTLIST ejemplo color CDATA #REQUIRED>
]>

    <ejemplo color="" />
    <ejemplo color="amarillo" />
    <ejemplo color="azul marino #000080" />
    <ejemplo />                                       <!-- ERROR: falta o atributo "color" -->
  • #IMPLIED: o atributo no es obligatorio e no se especifica ningún valor predeterminado. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo EMPTY>
  <!ATTLIST ejemplo color CDATA #IMPLIED>
]>

    <ejemplo />
    <ejemplo color="" />
    <ejemplo color="amarillo" />
    <ejemplo color="azul marino #000080" />
  • #FIXED valor: o atributo tiene un valor fijo. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo EMPTY>
  <!ATTLIST ejemplo color CDATA #FIXED "verde">
]>

    <ejemplo />
    <ejemplo color="verde" />
    <ejemplo color="" />                              <!-- ERROR: o atributo "color" no tiene o valor "verde" -->
    <ejemplo color="amarillo" />                      <!-- ERROR: o atributo "color" no tiene o valor "verde" -->
  • valor: o atributo tiene un valor predeterminado. 
    <!DOCTYPE ejemplo [
  <!ELEMENT ejemplo EMPTY>
  <!ATTLIST ejemplo color CDATA "verde">
]>

    <ejemplo />
    <ejemplo color="" />
    <ejemplo color="amarillo" />
    <ejemplo color="verde" />
