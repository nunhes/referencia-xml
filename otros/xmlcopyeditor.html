Conseguir XML Copy Editor

XML Copy Editor es un editor de documentos XML libre (GPL 2.0) e multiplataforma cuya páxina web es https://xml-copy-editor.sourceforge.io/.

La última versión disponible actualmente (febreiro de 2022) es a versión XML Copy Editor 1.3.0.0 (del 19 de agosto de 2020). Enlace de descarga para Windows (32 bits): XML Copy Editor 1.3.0.0 (17,8 MB).

Nota: En caso de que esta versión dé problemas, se puede utilizar a versión XML Copy Editor 1.2.1.3 (del 6 de setembro de 2014). Enlace de descarga para Windows (32 bits): XML Copy Editor 1.2.1.3 (9,2 MB).

En cdlibre.org hay unha sección dedicada a editores XML libres, con información detallada sobre as últimas versións publicadas para Windows.

Instalar XML Copy Editor

Una vez descargado o instalador de XML Copy Editor, haciendo doble clic en él se inicia a instalación.

  1. La primera pantalla anuncia que se va a instalar XML Copy Editor e indica a versión que se va a instalar. Para continuar, haga clic no botón "Next".

  2. La segunda pantalla muestra a licencia de XML Copy Editor (GPL 2 o posterior) e solicita al usuario su conformidad. Para continuar, marque o botón radio "I accept the agreement" e haga clic no botón "Next".

  3. La tercera pantalla pregunta a carpeta de instalación donde se instalará o programa. Por omisión, XML Copy Editor se instala en C:\Archivos de programa (x86)\XML Copy Editor. Para continuar, haga clic no botón "Next".

  4. La cuarta pantalla pregunta o nombre da carpeta no menú Inicio. Por omisión, a carpeta se llama XML Copy Editor. Para continuar, haga clic no botón "Next".

  5. La quinta pantalla pregunta si se crea un icono de acceso directo no escritorio, si se quiere instalar XML sin privilegios de administrador e si se asocian a XML Copy Editor as extensións de archivos que XML Copy Editor es capaz de manejar. Para continuar, haga clic no botón "Next".

  6. La sexta pantalla muestra todas as opcións elegidas nas pantallas anteriores e solicita confirmación para iniciar a instalación. Para continuar, haga clic no botón "Install".

  7. A continuación se iniciará o proceso de instalación. La instalación de XML Copy Editor puede durar unos minutos.

  8. Una vez completada a instalación, se mostrará a pantalla final. Haciendo clic en no botón "Finish" se cerrará o programa de instalación.

Interface e configuración de XML Copy Editor

  • O interface de XML Copy Editor es similar al de calquera editor de texto sin formato.

  • Al abrir XML Copy Editor o editor muestra a plantilla dun documento XML con a declaración XML. Si se quiere crear un documento XML desde cero, se puede elegir o menú Fichero > Nuevo... e elegir a opción Documento XML (*.xml), como muestra a imagen seguinte:

  • O menú Herramientas > Opcións > Editor permite configurar o comportamiento de XML Copy Editor al editar archivos XML. Personalmente, yo prefiero desmarcar a casilla "Siempre insertar etiqueta de cerrado".

  • En o menú Herramientas > Opcións > General se puede desactivar a opción "Guardar marca de orden de bytes UTF-8", xa que a marca de orden de bytes (BOM) no tiene mucho sentido en UTF-8, puesto que en UTF-8 no existe o problema do orden dos bytes en os caracteres que se codifican con varios caracteres.

    Nota: En a versión XML Copy Editor 1.2.0.7 e anteriores se podía elegir o navegador predeterminado de XML Copy Editor, independientemente do navegador predeterminado do sistema. Haciendo clic en Buscar se debía elegir o ejecutable do navegador.

Juego de caracteres

La declaración xml indica o xogo de caracteres do documento. O xogo de caracteres que se utiliza en este curso es UTF-8:

<?xml version="1.0" encoding="UTF-8"?>

Se pueden utilizar outros xogos de caracteres, como ISO-8859-1 (Europeo occidental):

<?xml version="1.0" encoding="ISO-8859-1"?>

Es importante que o xogo de caracteres que aparece na declaración sea o xogo de caracteres en que realmente está guardado o documento, porque si no o procesador XML puede tener problemas leyendo o documento.

XML Copy Editor tiene en cuenta o xogo de caracteres indicado na declaración. Si se modifica a declaración, al guardar o documento se guarda no xogo correspondiente. Pero hay que tener en cuenta que outros editores, como o bloc de notas de Windows, no lo hace.

Comprobar que un documento está ben formado

Para comprobar si un documento está ben formado, se puede elegir o menú XML > Comprobar Bien-Formado, hacer clic no botón correspondiente, o pulsar a tecla F2.

  • En caso de que o documento esté ben formado, o programa lo indica na parte inferior da pantalla, como muestra a imagen seguinte:

  • En caso de que o documento no esté ben formado, o programa lo indica na parte inferior da pantalla, como muestra a imagen seguinte. O mensaje inferior explica o tipo de error detectado e a línea no que se ha detectado (no ejemplo, as etiquetas da línea 3 no coinciden). Si o documento contiene varios errores, sólo se muestra uno de ellos, por lo que unha vez corregido un error es necesario repetir a comprobación hasta que o documento esté ben formado.

Comprobar que un documento es válido

Para comprobar si un documento es válido, se puede elegir o menú XML > Validar > DTD/XML Schema, hacer clic no botón correspondiente, o pulsar a tecla F5.

  • En caso de que o documento sea válido, o programa lo indica na parte inferior da pantalla, como muestra a imagen seguinte:

  • En caso de que o documento no sea válido, o programa lo indica na parte inferior da pantalla, como muestra a imagen seguinte. O mensaje inferior explica o tipo de error detectado e a línea no que se ha detectado (no ejemplo, que a etiqueta contiene texto cuando a DTD lo define vacío). Si o documento contiene varios errores, sólo se muestra uno de ellos, por lo que unha vez corregido un error es necesario repetir a validación hasta que o documento sea válido.

Enlazar unha folla de estilo CSS

  • XML Copy Editor no incluye opcións de menú relacionadas con o enlace a folla de estilo CSS, por lo que es necesario escribir manualmente a instrucción de procesamento.

  • La folla de estilo puede crearse con calquera editor de texto sin formato o con o propio XML Copy Editor.

    Al crear un nuevo documento, XML Copy Editor no ofrece a posibilidad de crear unha folla de estilo css, pero se puede crear un nuevo documento XML, guardarlo con o nombre e extensión deseados (no ejemplo, estilo.css), borrar a declaración XML e escribir a folla de estilo. Para que se coloree o código, puede ser necesario recargar o documento (mediante o menú Fichero > Recargar.
