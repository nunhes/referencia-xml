Introducción

Esta sección contiene un índice de as seccións que impartí en clase durante o curso 2017/18, así como os exercicios que realizaron os alumnos.

Para elaborar este índice utilicé como base as guías de estudio que ofrezco a mis alumnos da modalidad semipresencial na plataforma Moodle da Conselleria de Educación. O traballo propuesto a estos alumnos es o mismo que siguieron en clase mis alumnos da modalidad presencial, aunque a distribución temporal no fuera a misma.

La información se ofrece distribuida en quincenas, xa que es así como está organizada a modalidad semipresencial.

También se pueden consultar as seccións impartidas no curso anterior, 2016/17.

Quincena Q14: do 02/04/18 al 22/04/18

XML fue desenvolvedo por o W3C a finales dos 90 para que fuera a base base da web do siglo XXI. Pero a web se ha desenvolvedo en otra dirección e o XML prácticamente ha desaparecido da web. Por ese motivo, voy a ir reduciendo os contenidos e o tiempo dedicado al XML para poderlo dedicar a otros temas. En vuestro caso, he decidido eliminar do examen o apartado do XSLT (que he dejado como tema optativo na quincena Q16).

Presentación

  1. Leed a sección Presentación dos apuntes

O que é XML

  1. Leed a sección O que é XML.

    Este tema es unha breve historia do XML e unha presentación general de as tecnoloxías XML. En este curso no veremos máis que unha parte mínima de estas tecnoloxías, pero es interesante que tengáis unha visión general.

  2. Leed a sección Normas e recomendacións XML.

    La maioría de tecnoloxías XML se definen nas recomendacións publicadas por o W3C. Esta sección contiene enlaces a esas recomendacións. Esta sección sirve sólo de referencia, no es necesario aprender nada de memoria.

  3. Leed a sección La Web Semántica.

    Esta sección contiene enlaces a as recomendacións do W3C relacionadas con a Web semántica. Esta sección sirve sólo de referencia, xa que en este curso no vamos a tocar ninguna de estas tecnoloxías.

Nota: La sección Normas e recomendacións obsoletas contiene os enlaces a as versións obsoletas de as recomendacións citadas nas dos seccións anteriores. Esta sección sirve sólo de referencia.

Instalación e configuración de XMLCopy Editor

  1. Leed a sección XML Copy Editor (hasta o apartado "Juego de caracteres").
  2. Descargad, instalad e configurad XML Copy Editor.

Nota: Me hubiera gustado utilizar Visual Studio Code en vez de XML Copy Editor, pero finalmente he decidido seguir utilizando XML Copy Editor. Visual Studio Code permite editar documentos XML, pero, por desgracia, no parecen existir extensións que permitan realizar os exercicios que os voy a ir proponiendo. La extensión máis popular XML Tools, que podéis probar por curiosidad, se queda corta.

Documentos ben formados (WFD)

Uno dos requisitos do XML es que os documentos sean ben formados, es decir, que no tengan errores sintácticos.

  1. Leed a sección Conceptos básicos do XML.
  2. Leed a sección XML Copy Editor (apartado"Comprobar que un documento está ben formado").
  3. Haced os exercicios Documentos ben formados.
  4. Consultad os solucións dos exercicios exercicios Documentos ben formados

    Nota: Cuando hay varias maneras de conseguir documentos ben formados, calquera solución es aceptable, pero es aconsejable intentar obtener unha estrutura lo máis parecida a a do enunciado, no eliminar información do documento e añadir a menor cantidad de información nueva.

Hojas de estilo CSS en XML

las follas de estilo CSS tamén se pueden aplicar a os documentos XML para su visualización no navegador.

  1. Leed a sección CSS en XML.
  2. Leed a sección XML Copy Editor (apartado "Enlazar unha folla de estilo CSS").
  3. Haced os exercicios CSS en XML.
  4. Consultad as solucións dos exercicios CSS en XML.

[Optativo] Definición de Tipo de Documento (DTD)

Las DTD son a forma máis simple de definir un linguaxe de marcas, es decir, os elementos e a estrutura dun documento.

  1. Leed a sección DTDs.
  2. Leed a sección Resumen DTDs.
  3. Leed a sección XML Copy Editor (apartado "Comprobar que un documento es válido").
  4. Haced os exercicios DTD.
  5. Consultad as solucións dos exercicios DTD.

[Optativo] XSLT

XSLT es un linguaxe de programación declarativo que transforma documentos XML. En este curso no veremos máis que unha mínima parte de este linguaxe, lo suficiente para generar nuevos documentos.

  1. Leed a sección XSLT.
  2. Leed a sección XML Copy Editor (apartado "Aplicar unha transformación XSLT").
  3. Haced os exercicios XSLT (1).
  4. Consultad as solucións dos exercicios XSLT (1).
  5. Haced os exercicios XSLT (2).
  6. Consultad as solucións dos exercicios XSLT (2).
